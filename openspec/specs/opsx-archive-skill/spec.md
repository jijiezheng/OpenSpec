# OPSX Archive Skill 规范

## 目的

定义 `/opsx:archive` 技能的预期行为，包括就绪检查、规格同步提示、归档执行和用户面向输出。

## 需求

### 需求：OPSX Archive 技能

系统应提供 `/opsx:archive` 技能，用于在实验性工作流中归档已完成的变更。

#### 场景：归档所有产物完成的变更

- **当** 代理执行 `/opsx:archive` 并指定变更名称时
- **并且** schema 中的所有产物都完成
- **并且** 所有任务都完成
- **那么** 代理将变更移动到 `openspec/changes/archive/YYYY-MM-DD-<name>/`
- **并且** 显示带有归档位置的成功消息

#### 场景：变更选择提示

- **当** 代理执行 `/opsx:archive` 而不指定变更时
- **那么** 代理提示用户从可用变更中选择
- **并且** 仅显示活动变更（排除 archive/）

### 需求：产物完成检查

技能在归档前应使用产物图检查产物完成状态。

#### 场景：未完成产物警告

- **当** 代理检查产物状态时
- **并且** 一个或多个产物的状态不是 `done`
- **那么** 显示列出未完成产物的警告
- **并且** 提示用户确认继续
- **并且** 如果用户确认则继续

#### 场景：所有产物完成

- **当** 代理检查产物状态时
- **并且** 所有产物的状态都是 `done`
- **那么** 不带警告继续

### 需求：任务完成检查

技能在归档前应从 tasks.md 检查任务完成状态。

#### 场景：发现未完成的任务

- **当** 代理读取 tasks.md 时
- **并且** 发现未完成的任务（标记为 `- [ ]`）
- **那么** 显示显示未完成任务计数的警告
- **并且** 提示用户确认继续
- **并且** 如果用户确认则继续

#### 场景：所有任务完成

- **当** 代理读取 tasks.md 时
- **并且** 所有任务都完成（标记为 `- [x]`）
- **那么** 不带任务相关警告继续

#### 场景：没有任务文件

- **当** tasks.md 不存在时
- **那么** 不带任务相关警告继续

### 需求：规格同步提示

技能应在归档前提示同步增量规格（如果存在规格）。

#### 场景：增量规格存在

- **当** 代理检查增量规格时
- **并且** 变更中存在带有规格文件的 `specs/` 目录
- **那么** 提示用户："This change has delta specs. Would you like to sync them to main specs before archiving?"
- **并且** 如果用户确认，执行 `/opsx:sync` 逻辑
- **并且** 无论同步选择如何，继续归档

#### 场景：没有增量规格

- **当** 代理检查增量规格时
- **并且** 不存在 `specs/` 目录或没有规格文件
- **那么** 不带同步提示继续

### 需求：归档过程

技能应将变更移动到带日期前缀的存档文件夹。

#### 场景：成功归档

- **当** 归档变更时
- **那么** 如果不存在则创建 `archive/` 目录
- **并且** 使用当前日期生成目标名称 `YYYY-MM-DD-<change-name>`
- **并且** 将整个变更目录移动到存档位置
- **并且** 在归档的变更中保留 `.openspec.yaml` 文件

#### 场景：存档已存在

- **当** 目标存档目录已存在时
- **那么** 失败并显示错误消息
- **并且** 建议重命名现有存档或使用不同日期

### 需求：技能输出

技能应提供关于归档操作的清晰反馈。

#### 场景：同步后归档完成

- **当** 同步规格后归档完成时
- **那么** 显示摘要：
  - 已同步的规格（来自 `/opsx:sync` 输出）
  - 变更归档到的位置
  - 使用的 schema

#### 场景：不同步归档完成

- **当** 不同步规格归档完成时
- **那么** 显示摘要：
  - 注意规格未同步（如果适用）
  - 变更归档到的位置
  - 使用的 schema

#### 场景：带警告的归档完成

- **当** 归档完成但产物或任务未完成时
- **那么** 包含关于什么未完成的注释
- **并且** 建议检查归档是否是有意的
